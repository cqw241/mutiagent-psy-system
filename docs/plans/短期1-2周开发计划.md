  ## GitHub Flow Procedure
  每个分支严格执行同一流程：

  1. 确认 `main` 干净并同步：`git status --short --branch`，必要时 `git fetch origin`。
  2. 从 `main` 新建分支：`git checkout -b feature/<task-name> main`。
  3. 先写失败测试，再写最小实现；每个行为变更必须先看到对应测试失败。
  4. 运行该分支最小验证命令，再运行相关 smoke。
  5. 使用 Conventional Commits 提交。
  6. 推送分支：`git push -u origin feature/<task-name>`。
  7. 创建 PR 到 `main`，PR 描述包含变更范围、测试结果、风险和回滚方式。
  8. PR 合入后更新本地 `main`，再切下一个任务分支。

  ## Test Plan
  - 告警事件：`pytest -q tests/test_alert_service.py tests/test_alert_event_store.py tests/test_chat_api.py`
  - WebSocket 契约：`pytest -q tests/test_ws_chat.py tests/test_chat_api.py`
  - 风险评测：`pytest -q tests/test_risk_eval_cases.py`，并运行 `python3 scripts/run_risk_eval.py --mode mock --output-json /tmp/risk_eval.json`
  - 前端逻辑：`pnpm --dir frontend run test:node`
  - 前端构建：`pnpm --dir frontend run lint`、`pnpm --dir frontend run build`
  - 全量 smoke：`PYTHON_BIN=/path/to/env/python bash scripts/ci_check.sh`
